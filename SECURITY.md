# Security Policy

## Supported Versions
Only the latest major version (v1.x) receives security updates.

## Reporting a Vulnerability
If you discover a security vulnerability within this project, please DO NOT open a public issue. Instead, send an email to Rishvin Reddy or use GitHub's private vulnerability reporting feature.

## Response Timeline
We aim to acknowledge receipt of vulnerability reports within 48 hours and provide an initial assessment within 5 business days.

## Scope
The scope of this security policy covers the n8n automation workflows, the YAML compilation scripts, and the provided PostgreSQL database schema. External security tools (Nmap, Nuclei, Trivy) are maintained by their respective authors and should be reported upstream.

## Responsible Use
This project is an automation platform that performs active security testing (e.g., Nmap, Nuclei). You must ONLY use this software against systems and assets you own or are explicitly authorized to assess. Unauthorized use of this tool against third-party systems is strictly prohibited.
